Project overview

A custom-built website for woodworking professional Bridget J. Duffy with three equal goals: portfolio showcase, direct shop sales, and commission inquiry generation. Target audience is both local community members and broader online shoppers.

Custom Next.js build Portfolio + shop + commissions Local + online audience Mobile-first Hobbit-hole aesthetic
Brand & aesthetic direction

Warm Artisan Editorial with Hobbit-hole Coziness — The aesthetic evolved from a generic "rustic artisan" brief into something more specific and memorable: the warmth of a hobbit hole. Plaster-white walls, honey timber beams, sage green doors, and golden afternoon light. Cozy, handcrafted, magical without being kitsch.

The brand name Bridget J. Duffy is used in full as the wordmark — personal, trustworthy, maker-forward.

Navigation decisions
◆
Top navigation with hamburger on mobile — Chosen over left side tab bar. Gives hero full screen width on mobile, which matters for photography-forward layouts. Side tab bar was ruled out as it feels app-like and steals 52px of content width.
◆
Active page indicator — Current page link gets full white color with a honey amber (#C8963C) underline. Inactive links are muted sage (#C8E0B0).
◆
Persistent "Request commission" CTA in nav — Honey amber button, top-right corner on all pages. Never hidden.

Component & UI decisions
◆
Piece badge system — Four states: "New" (honey amber), "In stock" (foliage green), "Only 1 left" (aged timber, creates gentle urgency), "Sold" (dark heartwood). Sold cards have struck-through price and "Notify me" CTA.
◆
Portfolio lightbox — Full overlay on piece click. Image carousel with dot indicators, full specs panel (wood, dimensions, finish, lead time), two CTAs: "Commission something similar" and "View more photos".
◆
Shop sidebar filters — Wood type (checkbox), price range (slider), availability (checkbox), finish (color swatches). Sidebar is 200px wide, content takes remaining width.
◆
Persistent cart bar — Dark heartwood bar showing item count, total, and thumbnail previews. Sits below the shop filter toolbar. Always visible while shopping.
◆
Portfolio featured piece — First piece in portfolio grid spans 2 columns at 16:9 aspect ratio. Gives hero work extra visual weight.
◆
Section rhythm — Homepage alternates between cream (#FAF5EA), plaster (#F0E6C8), and straw (#EDE0B8) backgrounds to create visual separation without borders. Green section for Instagram embed.
◆
Commission process strip — Dark green (#5A7A3C) four-step strip on homepage and commissions page. Steps: Consult → Design → Build → Deliver.
◆
Hover states — Portfolio pieces: scale(1.02) with dark overlay revealing name and specs. Shop cards: translateY(-2px). Buttons: wood-burn darkening effect.
Tech stack decisions
◆
Framework: Next.js (custom build) — Chosen over Webflow for flexibility and scalability. Works fully on mobile via responsive CSS — no separate mobile app needed.
◆
No native mobile app — Ruled out. App store fees ($99/yr Apple, $25 Google one-time) and 30% revenue cut on in-app purchases make it uneconomical for a shop. Stripe's ~2.9% fee via website is far cheaper.
◆
Payments: Stripe — For shop checkout. Shopify Buy Button as alternative if CMS-managed inventory is preferred.
Next.js Stripe Cloudinary (images) LightWidget (Instagram) Mailchimp (email list) Netlify Forms (commission form)
Next steps
01
Complete remaining page mockups
Three pages still need mockups before moving to implementation: Commissions (multi-step form, process timeline, FAQ), About the Maker (story, workshop photos, philosophy), and Contact (form, location, map embed). Build in this order — commissions is the most complex.
02
Final homepage pass after all pages are done
The homepage references every other page (featured pieces, shop preview, commission strip, testimonials). Do one final review pass once all inner pages are locked in — inner page decisions often feed back and refine the homepage.
03
Gather photography
8–12 high-quality photos of finished pieces are the single most important asset. Every hero image placeholder in the mockups is a slot for real photography. Consider hiring a photographer for a half-day shoot — natural light, workshop context in background, warm and slightly desaturated treatment.
04
Write copy for all pages
Key copy needed before build: maker story (2–3 paragraphs, About page), hero tagline (currently "Made by hand. Built to last generations." — confirm or refine), commission process descriptions, FAQ answers, and product descriptions for initial shop inventory.
05
Define initial shop inventory
List all available pieces with: name, wood species, dimensions, price, finish, and lead time (for commissions). This data structure will inform how the Next.js shop is built.
06
Collect 3–5 testimonials
Reach out to past clients for short quotes. Ideal format: quote, client name (first + last initial), and project type. Needed before launch — they appear on the homepage and commissions page.
07
Move to Claude Code for implementation
Once all mockups are approved and copy/photography is in hand, hand off to Claude Code with this design decisions doc, the final palette, and all mockups as reference. It can scaffold the entire Next.js project — component structure, CSS variables, page layouts — from the start. Recommended trigger: all 6 page mockups approved + palette confirmed.
08
Set up Stripe and third-party integrations
Stripe account setup, Cloudinary for image hosting/optimization, LightWidget for Instagram embed, Mailchimp for newsletter, Netlify Forms or similar for the commission request form. These can be configured in parallel with the build.
